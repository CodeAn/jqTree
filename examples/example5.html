Example 5 - Load nodes on demand from the server

This examples loads data on demand in the tree.before_open event.

html

In this example, the /nodes/ url must return data for the first level of nodes. 

<div id="tree1" data-url="/nodes/"></div>

One detail: as this is the first level of data, the nodes have no children property. To indicate that these nodes are folders, the property is_folder is set to true: 

{
  "label": "Saurischia",
  "id": 1,
  "is_folder": true
}

javascript

When the user opens a folder, its children are loaded from the server. This is done in the tree.before_open event. This event gives you the opportunity to delay the opening of the folder. 

var $tree = $('#tree1');

$tree.tree();

$tree.bind('tree.before_open', function(e) {
  var node = e.node;

  // This node is folder. But does it have children? If not, load them.
  if (! node.hasChildren()) {
    var $folder = $(e.node.element);
    $folder.addClass('loading');

    $.getJSON(
        '/nodes/' + node.id,
        function(data) {
          $folder.removeClass('loading');

          // Load the data
          $tree.tree('loadData', data, node);

          // Open the folder
          e.do_open();
        }
    );

    // Delay the opening of the folder
    e.preventDefault();
});
